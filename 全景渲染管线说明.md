# 全景画面渲染管线与数据流说明

## 一、整体架构总览

本项目是基于 **React + React Three Fiber (R3F) + Three.js + Zustand** 构建的全球街景漫游应用。从地点切换到全景画面呈现，数据流经 **7 个层级**，渲染经过 **6 个阶段**。

```
┌─────────────────────────────────────────────────────────────────┐
│  层级1：UI 交互层  (LocationListSidebar / Header / Map / Random) │
│         ↓ 触发 handleSelectLocation / handleRandom               │
├─────────────────────────────────────────────────────────────────┤
│  层级2：页面状态层  (Home.tsx useState)                          │
│         ↓ setCurrentLocation / setIsTransitioning / setIsLoading │
├─────────────────────────────────────────────────────────────────┤
│  层级3：全局 Store 层  (useTravelStore / useEditorStore)         │
│         ↓ addVisitedLocation / 订阅 weather/fov/filter          │
├─────────────────────────────────────────────────────────────────┤
│  层级4：R3F Canvas 容器层  (StreetViewer / DualPanoramaViewer)   │
│         ↓ <Canvas> 创建 WebGLRenderer / Scene / Camera          │
├─────────────────────────────────────────────────────────────────┤
│  层级5：场景内容层  (PanoramaSphere / CameraController / Stars)  │
│         ↓ TextureLoader / OrbitControls / 粒子系统              │
├─────────────────────────────────────────────────────────────────┤
│  层级6：后处理叠加层  (WeatherParticles / OverlayLayer / CSS Filter)│
│         ↓ useFrame 逐帧更新 / DOM 叠加水印贴纸                   │
├─────────────────────────────────────────────────────────────────┤
│  层级7：音频管线层  (useAmbientSound → Web Audio API)           │
└─────────────────────────────────────────────────────────────────┘
```

### 阶段 3：PanoramaSphere 纹理加载（核心渲染）

[PanoramaSphere.tsx:L12-L72](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/PanoramaSphere.tsx#L12-L72) 是全景渲染的核心。

#### 3.1 几何体构建

```jsx
<sphereGeometry args={[500, 60, 40]} />
```
- 半径 **500** 单位的球体（足够大以包含相机）
- 宽度分段 60，高度分段 40（平衡精度与性能）

```jsx
<meshBasicMaterial side={THREE.BackSide} />
```
- `BackSide` 渲染**内侧**（相机在球心看向内壁）
- `MeshBasicMaterial` 不受光照影响，保证全景图原样输出

#### 3.2 纹理加载流程（useEffect 监听 panoramaUrl 变化）

```
[location.panoramaUrl 变化]
        ↓
setLoaded(false)          → 标记开始加载
        ↓
TextureLoader.load()      → 异步加载 equirectangular 全景图
        ↓ 成功
texture.colorSpace = THREE.SRGBColorSpace  → 保证颜色正确
texture.needsUpdate = true                → 标记 GPU 更新
material.map = texture                    → 绑定到材质
material.color.set(0xffffff)              → 白色底色（显示纹理）
material.needsUpdate = true
setLoaded(true)                           → 加载完成
onLoad?.()                                → 通知父组件 → Home 的 handleSceneReady → setIsLoading(false)
        ↓ 失败
material.color.set(0x00aaff)              → 蓝色兜底
setLoaded(true)
onLoad?.()                                → 依然触发完成回调
```

#### 3.3 加载期间的视觉效果
[PanoramaSphere.tsx:L17-L21](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/PanoramaSphere.tsx#L17-L21)：
纹理加载完成前（`loaded=false`），球体会以 0.05 rad/帧的速度缓慢自转，搭配蓝色初始色，形成类似"传送门"的视觉过渡。

### 阶段 4：CameraController 相机初始化与控制

[CameraController.tsx:L43-L140](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/CameraController.tsx#L43-L140) 负责视角控制。

#### 4.1 球面坐标系统

项目使用自定义的 **Heading（方位角）/ Pitch（俯仰角）** 系统，而非直接操作欧拉角：

| 术语 | 含义 | 范围 |
|------|------|------|
| Heading | 水平旋转角（左右看） | -180° ~ 180° |
| Pitch | 垂直俯仰角（上下看） | -75° ~ 75°（受极角限制） |

**核心转换函数**：

- `setCameraFromSpherical()` [L32-L41](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/CameraController.tsx#L32-L41)：Heading/Pitch → 三维坐标
  ```
  phi   = (90 - pitch)° 转弧度  → 极角（从 Y 轴正方向往下）
  theta = heading° 转弧度       → 方位角（绕 Y 轴）
  radius = 400
  x = radius * sin(phi) * cos(theta)
  y = radius * cos(phi)
  z = radius * sin(phi) * sin(theta)
  camera.lookAt(0, 0, 0)  → 始终看向球心
  ```

- `getSphericalFromCamera()` [L24-L30](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/CameraController.tsx#L24-L30)：逆向计算

#### 4.2 初始化流程

`locationKey` 变化时触发 [L77-L83](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/CameraController.tsx#L77-L83)：
```

    ↓
OrbitControls.update()
    ↓

```

每个地点自带初始视角（来自 [locations.ts](file:///Users/tog/Desktop/code/solo/xyj-127/src/data/locations.ts)）：

| 地点 | initialHeading | initialPitch | 说明 |
|------|---------------|-------------|------|
| 香榭丽舍大街 | 0° | 0° | 正前方平视 |
| 曼哈顿第五大道 | 90° | 5° | 右前方微仰看摩天楼 |
| 新宿街头 | 45° | 3° | 右前 45° |
| 大峡谷南缘 | 135° | -8° | 右后方向下俯瞰 |
| 冰岛极光之夜 | 0° | 25° | 正前方仰看极光 |

#### 4.3 交互控制模式

| 模式 | OrbitControls enableRotate | 数据源 | 适用场景 |
|------|---------------------------|--------|---------|
| 自由交互 | true | 用户鼠标拖拽 | 默认模式 |
| 外部受控（双屏同步） | false | `controlledHeading/Pitch` | DualPanoramaViewer 的 sync 模式 |

#### 4.4 自动旋转与 FOV

- **自动旋转**：从 `useEditorStore` 订阅 `autoRotate` 和 `autoRotateSpeed`
- **FOV 动态调整**：从 `useEditorStore` 订阅 `fov`，修改 `camera.fov` 并 `updateProjectionMatrix()`
- **极角限制**：`minPolarAngle=30°`, `maxPolarAngle=150°`（即 Pitch 限制在 ±60° 左右）

#### 5.2 天气粒子系统
[WeatherParticles.tsx](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/WeatherParticles.tsx) 采用 **5 层职责架构**：

```
┌─ 配置层：WEATHER_BASE_CONFIG（每种天气的 maxCount/size/speed/雾参数）
├─ 强度计算层：calcDynamicParams() 根据 intensity (0~1) 计算可见粒子数/速度/雾密度
├─ 粒子生成层：球壳均匀分布（半径 30~340），避免相机穿模
├─ 粒子运动层：useFrame 逐帧更新位置 + 边界重生（雨下落、雪飘动、雾漂移、沙横飞）
└─ 渲染层：<points> + PointsMaterial（AdditiveBlending 加法混合更真实）
```

**天气配置对比**：

| 天气类型 | 最大粒子数 | 速度基准 | 雾颜色 | 混合模式 |
|---------|-----------|---------|--------|---------|
| rain | 6000 | 110 | 0x556677 | Additive |
| snow | 5000 | 10 | 0x9eb4c8 | Additive |
| fog | 800 | 0.6 | 0xc8d0d8 | **Normal**（体积感） |
| sand | 7000 | 65 | 0xd4a574 | Additive |

**天气切换 useEffect** [L298-L346](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/WeatherParticles.tsx#L298-L346)：
- 重新计算 `visibleCount = 0.2 + intensity * 0.9` 比例的粒子
- 初始化位置、速度、颜色、尺寸
- 超出可见数量的粒子放到 y=-10000（视锥外不可见）

#### 5.3 雾效
`FogExp2`（指数雾）附加到 `scene.fog`，密度随 intensity 动态调整。

## 四、双屏对比模式（DualPanoramaViewer）特殊说明

[DualPanoramaViewer.tsx](file:///Users/tog/Desktop/code/solo/xyj-127/src/components/DualPanoramaViewer.tsx) 创建 **两个独立的 R3F Canvas**，左右各占 50% 宽度。

### 同步模式下的数据流

```
[左 Canvas OrbitControls 用户拖拽]
        ↓
handleLeftCameraChange(heading, pitch)
        ↓
setLeftHeading / setLeftPitch  (DualPanoramaViewer 的本地 state)
        ↓
viewMode === 'sync' && syncSource === 'left'
        ↓
setRightHeading(heading), setRightPitch(pitch)
        ↓
右 PanoramaView 的 controlledHeading / controlledPitch props 更新
        ↓
右 CameraController.isControlled = true
        ↓ [L68-L75 CameraController]
setCameraFromSpherical(camera, controlledHeading, controlledPitch)
OrbitControls.update()
enableRotate = false  → 右侧禁止直接拖拽
```

### 独立模式
左右两侧 `viewMode === 'independent'`，各自 `controlledHeading/Pitch = undefined`，两侧都可独立交互。

## 五、关键设计亮点

1. **600ms 过渡假动画 + 真实纹理异步加载**：用户感知流畅，不被网络 IO 阻塞
2. **球面坐标系统**：Heading/Pitch 抽象让"预设视角""双屏同步"等功能极其简洁
3. **球壳粒子分布**：天气粒子在半径 30~340 的球壳内生成，始终跟随相机位置，永远不会穿模
4. **5 层天气架构**：配置/计算/生成/运动/渲染解耦，新增天气类型只需扩展配置表
5. **双 Canvas 同步**：用受控模式简单实现了复杂的双视角联动
6. **Web Audio 程序化合成**：无需音频资源文件，通过 Oscillator + Filter + LFO 合成环境音
